Node.js开发的坑 1

发表于 2016-06-25

版本问题

常见的问题是，本地开发电脑可以用，当推到服务器上的时候却不能用，会抛出各种错误：

如下：

/phpstudy/www/SRX_DO_IN_BACKGROUND/node_modules/connect-mongo/src/index.js:3
const Promise = require('bluebird');
^^^^^
SyntaxError: Use of const in strict mode.
at Module.\_compile (module.js:439:25)
at Object.Module.\_extensions..js (module.js:474:10)
at Module.load (module.js:356:32)
at Function.Module.\_load (module.js:312:12)
at Module.require (module.js:364:17)
at require (module.js:380:17)
at Object.<anonymous> (/phpstudy/www/SRX_DO_IN_BACKGROUND/node_modules/connect-mongo/index.js:1:80)
at Module.\_compile (module.js:456:26)
at Object.Module.\_extensions..js (module.js:474:10)
at Module.load (module.js:356:32)

在本地不会出错，但是在阿里的云服务器上会报此错误，网上查询得知是node的版本不对，让我们看看阿里的node版本，默认是用

sudo yum install epel-release
sudo yum install nodejs

命令安装，版本是v0.10.42

解决方案换node版本

卸载

yum remove nodejs

下载nodejs

wget https://nodejs.org/dist/v4.4.4/node-v4.4.4.tar.gz

安装

tar zxvf node-v4.4.4.tar.gz
cd node-v4.4.4 & make

很难等。。.干点别的

make install

开始安装,一般很快

测试

node -v
